# Parallel Prep Checklist

A week-by-week checklist of tasks that should run **in parallel** with book reading and project building. These are the things people forget to do until it's too late — and then scramble to catch up.

---

## How to Use This Checklist

- This is a **parallel track**, not a main track. It should take 30–60 minutes per day maximum.
- During project-heavy months, reduce to 30 min/day.
- During book-heavy months, you can invest slightly more.
- Check off items as you complete them.

---

## Phase 1: Infrastructure (March 2026) — "Set It Up Right"

### Week 1 (Mar 11–17)
- [ ] Create/update GitHub profile with professional README
- [ ] Set up Weights & Biases account; run a test experiment
- [ ] Create `arc-system` repository with folder structure from architecture-overview.md
- [ ] Install ARC evaluation harness; run on a sample submission (all zeros — just verify it works)
- [ ] Create a private Notion/Obsidian notebook for paper notes
- [ ] Solve 10 ARC tasks manually on the [ARC Interface](https://arcprize.org/play); annotate which skills each requires

### Week 2 (Mar 18–24)
- [ ] Solve 20 more ARC tasks manually (total: 30)
- [ ] Write 2-paragraph summaries of the first 3 papers (Random Forests, SVM, AdaBoost)
- [ ] Watch 3Blue1Brown "Essence of Linear Algebra" playlist (3 videos)
- [ ] Set up Docker environment for Project 1
- [ ] LeetCode: solve 5 easy array problems

### Week 3 (Mar 25–31)
- [ ] Complete Chollet's *On the Measure of Intelligence* (Phase 1, paper #18) — read this early
- [ ] Solve 20 more ARC tasks manually (total: 50) — **non-negotiable before writing any code**
- [ ] Write 1-page reflection: "What makes ARC tasks hard for ML systems?"
- [ ] Watch 3Blue1Brown "Neural Networks" playlist (4 videos)
- [ ] LeetCode: 5 medium problems (linked lists, trees)

### Week 4 (Apr 1–7)
- [ ] Begin Project 1 code (features.py first)
- [ ] Open ARC-AGI leaderboard; identify top-3 public approaches; read their descriptions
- [ ] Set up experiment tracking for Project 1 in W&B
- [ ] Write Project 1 design doc (1 page) before coding
- [ ] LeetCode: 5 medium problems (graphs, BFS/DFS)

## Phase 2: Building Credibility (May–June 2026) — "Make Things Visible"

### May (Book 2 month)
- [ ] Post Project 1 code publicly on GitHub with a README that includes ARC-AGI-1 score
- [ ] Write a blog post / LinkedIn article about your ARC-Perceiver approach and results (even if score is modest)
- [ ] Join ARC-AGI Discord or community forum; introduce yourself
- [ ] Solve 20 more ARC tasks (total: 70) — focus on tasks your Project 1 system failed on
- [ ] LeetCode: 20 medium problems over the month
- [ ] Read Slot Attention paper (Locatello 2020) and implement it on CLEVR before building NeuralSolver

### June (Project 2 month)
- [ ] Post ARC-NeuralSolver v1 on GitHub when running (even with modest score)
- [ ] Upload model weights to HuggingFace (free, massive credibility signal)
- [ ] Write a 1-page "what I learned" after Project 2 — honest about failures
- [ ] LeetCode: 20 medium problems over the month (focus: dynamic programming)
- [ ] Create a demo: Gradio interface where users input an ARC grid pair and see NeuralSolver's prediction
- [ ] Start following 5–10 ARC researchers on Twitter/X

## Phase 3: Research Output (July–August 2026) — "Start Writing"

### July (Book 3 month)
- [ ] Begin workshop paper draft for Project 2 (even if rough)

- [ ] Set up Overleaf account; use NeurIPS or ICML workshop template
- [ ] Post custom BPE tokenizer for ARC grids as a standalone GitHub repo — this is useful to others
- [ ] LeetCode: 20 medium problems (string manipulation, dynamic programming)
- [ ] Identify 3–5 workshops relevant to your work (NeurIPS workshops, ICLR workshops)

### August (Project 3 month)
- [ ] Submit Project 2 paper to a workshop (even if it's not perfect — experience matters)
- [ ] Post ARC-Programmer on GitHub with complete documentation
- [ ] Upload fine-tuned LLM to HuggingFace with a model card
- [ ] Write a detailed blog post: "How to fine-tune a GPT-2 model to solve ARC tasks with a custom DSL"
- [ ] LeetCode: 20 medium/hard problems (system design prep starts)
- [ ] Update LinkedIn with Project 3 description and ARC-AGI score

## Phase 4: Job Preparation (September–October 2026) — "Get Hired"

### September (Book 4 month)
- [ ] Begin aggressive LeetCode: 10 problems/week (medium + hard)
- [ ] Identify 20 target companies; research each ML team (5–10 min each)
- [ ] Draft cold outreach template for ML researchers at target companies
- [ ] Attend 1–2 online ML events (virtual conferences, meetups)
- [ ] Build personal website with project portfolio (GitHub Pages or similar; 2–4 hours total)
- [ ] Request LinkedIn recommendations from any professional contacts
- [ ] Practice 30-second elevator pitch for your ARC work

### October (Project 4 month)
- [ ] Begin active job applications (10/week minimum)
- [ ] Submit ARC-Reasoner paper to workshop
- [ ] Do 5+ mock interviews (use Pramp, interviewing.io, or friends)
- [ ] Practice ML system design: recommendation systems, ML pipelines, distributed training
- [ ] Post ARC-Reasoner on GitHub; write comprehensive README with ARC-AGI-1 and ARC-AGI-2 scores
- [ ] Submit entry to ARC-AGI competition (if active)
- [ ] Follow up on all warm connections from September networking

## Ongoing Weekly Tasks (Every Week, All 8 Months)

- [ ] LeetCode: minimum 5 problems/week (even during heavy project weeks)
- [ ] Paper summaries: write 2-paragraph summary for each paper read
- [ ] Commit code: minimum 3 meaningful commits/week to public repos (demonstrates activity)
- [ ] Lab notebook: 5 minutes end-of-day noting what you learned, what failed, what surprised you
